# Trigger Evals

## Should Trigger

- Add a feature and keep the product contract honest.
- Fix this user-visible bug and update the expected behavior.
- Change this command output and make sure the claim and test match.
- Introduce the first Special specs while shipping this feature.

## Should Not Trigger

- Fix this Rust borrow-checker error.
- Summarize the latest release notes.
- Move code between modules without behavior change.
- Decide whether repeated code should be a pattern.
