# Trigger Evals
@implements SPECIAL.DOCUMENTATION.SKILLS.FALLBACK.USE_PROJECT_PATTERNS
@applies DOCS.SKILL_TRIGGER_EVAL_REFERENCE

## Should Trigger

- Add code that follows the same approach as the existing cache fill code.
- Review whether this implementation should match an existing approach.
- The patterns metrics say this file has a possible missing application.
- Decide whether this repeated UI component shape should be a pattern or a shared component.
- Update a pattern definition after refactoring its applications.
- Check why this code is done this way elsewhere in the repo.

## Should Not Trigger

- Define a product behavior spec.
- Verify whether one spec is honestly supported.
- Install bundled skills.
- Find planned product work.
- Format or lint code without changing implementation approach.
- Write a one-off comment explaining a local workaround.
